# Skill: README Excellence Pass

## Purpose
Ensure the project README is clear, complete, and inviting for new visitors.

## When to Use
- Before an open-source release or public announcement.
- After significant feature additions that change the project scope.

## Steps
1. Verify the README includes: project title, one-line description, screenshot or demo link.
2. Check for a "Getting Started" section with prerequisites, install, and run commands.
3. Confirm the tech stack is listed concisely.
4. Verify links to architecture docs, design system, and roadmap.
5. Ensure no em dashes are used; follow `docs-style.md` conventions.
6. Add a license badge and status badges (CI, deploy) if applicable.
7. Read the README as a newcomer: is it clear what the project does within 10 seconds?

## Key Files
- `README.md` - the project README.
- `.claude/rules/docs-style.md` - documentation style guide.
